Key Findings

Fully local Hunyuan3D-2 + Kijai ComfyUI pipeline built from scratch after all commercial photogrammetry options hit paywalls or produced unusable noise.
Custom_rasterizer_kernel successfully compiled on embedded Python 3.13 + CUDA 13.0 after repeated MSVC, Python.h, and nvcc version conflicts; final .pyd renaming + pycache flush was the only reliable fix for ModuleNotFound/PyInit errors.
Root cause of persistent blotching identified as noisy normal/position maps generated by Hy3DRenderMultiView from single-image geometry (confirmed via noiise102.png multi-view preview panels).
Maximum GPU/VRAM residency achieved (~98-99% utilization) by iteratively raising MAX_SAFE_PIXELS_GPU to 42M pixels while retaining try/except zero-tensor fallback hardening; this eliminated most precision loss from CPU offload.
2048×2048 base resize + 4x Foolhardy Remacri upscale wired post-bake is the optimal production combination for single-source inputs; render_size=768 in node 79 is the single highest-impact knob for smoothing maps.
Skills permanently gained: CUDA kernel compilation/debugging, tensor clamping before gather, VRAM profiling under embedded Python constraints, and turning every OOM into graceful safe-zero instead of crash.
Dogged pursuit of exotic local solution over paid tools produced transferable capability that directly fulfills the Bible’s AI-centric mandate.

Rejected / Dark-Horse Ideas

Paid photogrammetry (KIRI higher tiers, Tripo/Meshly, RealityScan): rejected on budget and noise grounds; early experiments confirmed they were not viable for Nanite-ready output.
CPU/RAM offload for large textures: tried extensively but rejected after visible quality loss in UV unwrap and final textures (user feedback “marginally better” only after GPU-first thresholds).
Lower render_size defaults (512): rejected after noisy map diagnosis; higher values (768+) repeatedly outperformed despite VRAM pressure.
Early Meshroom fallback: downloaded but never used once Hunyuan path proved viable; discarded to avoid diluting AI-centric focus.
Adding Canny ControlNet mid-pipeline: considered but rejected as it would break Kijai node compatibility without major wrapper surgery.
